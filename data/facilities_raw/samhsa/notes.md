# SAMHSA / findtreatment.gov source notes

- No public bulk CSV/JSON download of the FindTreatment.gov facility locator dataset was accessible without registering for API access (https://findtreatment.gov/api-request-form) or a Developer Guide-authenticated key. Attempted a guessed `exportCsv` endpoint directly via curl; it returned the HTML app shell rather than data (SPA-driven, no anonymous data endpoint).
- data.gov catalog entry "Mental Health Treatment Facilities Locator" was identified (https://catalog.data.gov/dataset/mental-health-treatement-facilities-locator) but not deep-fetched for a direct download link in this pass.
- As a result, SAMHSA-sourced records in facilities.json (`data_source: ["samhsa"]`) are based on general knowledge of named organizations known to appear in SAMHSA's locator for SF/San Mateo/Alameda counties (BAART, RAMS, Felton Institute, Instituto Familiar de la Raza, Swords to Plowshares, John George Psychiatric Pavilion, Cordilleras, Bonita House, Horizon Services, StarVista, Telecare), with source_url pointing to the locator homepage rather than a per-facility deep link, since deep links require a session-based search rather than a stable permalink.
- GAP: These SAMHSA-tagged records should be treated as lower-confidence placeholders pending either (a) API registration and a proper bulk pull, or (b) manual one-by-one confirmation via https://findtreatment.gov/locator search UI.
- Out-of-county fallback (San Mateo, Alameda) facilities are included per task instructions but are even lower-confidence/placeholder in nature (addresses are city-level only, not street-level, for several).
