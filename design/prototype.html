Domain Extractor & Reconciliation LLM
Reads longitudinal notes and extracts domain-level clinical state. Every value is bound to a verbatim quote, source note, and timestamp — nothing is asserted without a citation.
in: day0_psych_eval.txt + day2_psych_transcript.txt out: {"domain":"SI_intent","status":"not_reassessed", "quote":"...hoy no tiene esos pensamientos.", "source":"psych_transcript","time":"2026-07-12 09:50"}
Readiness Diff Engine deterministic
Compares patient readiness fields against a target facility's own declared intake_requirements. No language model in this step — the extractor supplies values, this engine only diffs them.
Decline Classifier LLM
Maps a facility's response into a fixed taxonomy — no_capacity / out_of_network / missing_docs / acuity_too_high / behavioral_exclusion — and proposes the matching next action instead of blind re-broadcast.
This system produces coordination signals for people to act on. It does not decide dispositions, and no flag is written to a permanent record without an explicit clinician or case-manager action.
Facility identities and locations shown throughout are real, public information. Bed counts, wait times, and decline behavior are simulated for demonstration and are not live data.
